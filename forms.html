Forms

We should aim to keep any form or data entry fields as simple as possible. If we can infer information such as location then do not ask the user to input it.

Always show the correct and most useful keyboard with the relevant fields. For instance, numerical input fields should always display the numeric specific keyboard.

Form Elements

Input field

Standard entry field with label directly above. Use whereever possible in this format.

Normal
Invalid

Input field inline-labels

Entry field with labels to the left when space is at a premium and the data fields are easily comparable.

Password Field

A special input field for password entry that allows the user to toggle whether or not to 'hide' their password as they type. In general if will be easier for the user to show the password by default as they type.

Product Search Field

A special input field that lets the user know with an icon that whatever they scan will be entered into this field and automatically submitted.

Stepped Control

For inputing numerical data, the addition of a stepped control for small adjustments of quantity so that the user can avoid inputing manually.

Radio Buttons

Radio Button List

For mutually exlusive selection of options within a list. Use instead of a segmented control when there are more than 3 options or longer titles on the options.

Checkboxes

Checkbox list

Use a checkbox list whenever a user can select more than one option from a list.
