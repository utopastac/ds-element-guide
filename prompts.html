Prompt and Notifications

Feeback from the system to the user is vital in building confidence in the system. Use prompts when the user is performing a potantially damaging action and there is no undo option. Use notifications to let users know something has happened or changed after their action.

Prompts

Do something prompt

Use when the user is about to change the state of a system irreversably. Generally the button options will be some form of 'cancel' or 'confirm'. Make the positive action match the task the user is trying to complete.

Thing happens

Thing has happened. Do thing to fix thing. Please fix the thing.

  • Cancel
  • Confirm

Confirm something prompt

Rarely used prompt to remind the user of what they have done or are about to do. Use for messaging change in applications or when there is nothing for the user to perform.

Thing happens

You must accept this thing. It is the best thing that ever happened.

  • Confirm

Notifications

Success Notification

Display when a user action has successfully been comitted to the server.

Success messages are often quite long and tell you the current state of the system.
